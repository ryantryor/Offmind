# Type-driven development in Python with mypy strict

Start with strict mode on new code - it's much easier than retrofitting. Use TypedDict for dict-shaped data, Protocol for duck-typed interfaces. Pyright (used by Pylance) is faster and stricter than mypy - prefer it for new projects.
